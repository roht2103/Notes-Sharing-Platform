╚══════════════════════════════════════════════════════════════════╝

🚨 PROBLEM: Render Free Tier doesn't allow Shell access to run:
   python manage.py createsuperuser

✅ SOLUTIONS: 3 ways to create admin user without Shell


═══════════════════════════════════════════════════════════════════
SOLUTION 1: AUTOMATIC CREATION DURING BUILD (EASIEST) ⭐
═══════════════════════════════════════════════════════════════════

The build.sh script now automatically creates an admin user during
deployment!

📝 HOW IT WORKS:
─────────────────────────────────────────────────────────────────
When you deploy to Render, build.sh runs and executes:
    python manage.py create_default_admin

This creates:
    Username: admin
    Password: admin123
    Email: admin@notesplatform.com

✅ STEPS:
1. Push your code to GitHub (build.sh is already updated)
2. Deploy to Render
3. Wait for build to complete
4. Check logs for "Default admin created successfully!"
5. Go to: https://your-app.onrender.com/admin
6. Login with: admin / admin123
7. ⚠️ CHANGE PASSWORD IMMEDIATELY!

🔒 SECURITY TIP: Change password after first login!
   Admin Panel → Click username → Change Password


═══════════════════════════════════════════════════════════════════
SOLUTION 2: CUSTOM ADMIN CREDENTIALS (SECURE) 🔐
═══════════════════════════════════════════════════════════════════

Set your own admin credentials using environment variables.

✅ STEPS:
1. Go to Render Dashboard → Your Web Service → Environment
2. Add these variables (click "Add Environment Variable"):

   Key: ADMIN_USERNAME
   Value: your_custom_username

   Key: ADMIN_EMAIL
   Value: your_email@example.com

   Key: ADMIN_PASSWORD
   Value: your_secure_password_here

3. Save and Render will auto-redeploy
4. Admin will be created with your custom credentials
5. Login at: https://your-app.onrender.com/admin

💡 ADVANTAGE: Secure admin credentials from day one!


═══════════════════════════════════════════════════════════════════
SOLUTION 3: WEB-BASED SETUP PAGE (ALTERNATIVE) 🌐
═══════════════════════════════════════════════════════════════════

Create admin through a web interface (if build.sh fails or you
prefer manual setup).

✅ STEPS:
1. Deploy your app to Render
2. Go to: https://your-app.onrender.com/setup/
3. Fill in the form:
   - Username
   - Email
   - Password (min 8 characters)
   - Confirm Password
4. Click "Create Admin Account"
5. Login at /admin with your credentials

📝 NOTES:
- Setup page only works when NO admin exists
- Automatically disabled after first admin is created
- Can be manually disabled with DISABLE_SETUP=True env variable

⚠️ If build.sh already created admin, /setup/ will redirect you.

═══════════════════════════════════════════════════════════════════
AFTER DEPLOYMENT: HOW TO CHECK IF ADMIN EXISTS
═══════════════════════════════════════════════════════════════════

Option A: Check Build Logs
1. Render Dashboard → Your Service → Logs
2. Look for: "Default admin created successfully!"
3. Or: "Admin user 'admin' already exists"

Option B: Try to Login
1. Go to: https://your-app.onrender.com/admin
2. Try: admin / admin123 (default)
3. Or try your custom credentials

Option C: Visit Setup Page
1. Go to: https://your-app.onrender.com/setup/
2. If admin exists: "Admin user already exists"
3. If no admin: Setup form appears


═══════════════════════════════════════════════════════════════════
TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

❌ "Admin user already exists" but I can't login
   ✅ Check build logs for which credentials were used
   ✅ Try default: admin / admin123
   ✅ Check if you set ADMIN_USERNAME/ADMIN_PASSWORD env vars

❌ Build.sh created admin but login fails
   ✅ Wait 30 seconds after deployment (cold start)
   ✅ Clear browser cache
   ✅ Try different browser
   ✅ Check /admin URL is correct (with trailing slash)

❌ /setup/ redirects me away
   ✅ Admin already exists (created by build.sh)
   ✅ Use default credentials: admin / admin123
   ✅ Or check your env vars for custom credentials

❌ Build failed, no admin created
   ✅ Check build logs for errors
   ✅ Use Solution 3: Visit /setup/ to create manually
   ✅ Verify build.sh has execute permissions (chmod +x)

❌ "DISABLE_SETUP is True" message
   ✅ Remove DISABLE_SETUP env variable
   ✅ Or admin was already created by build.sh


═══════════════════════════════════════════════════════════════════
SECURITY BEST PRACTICES
═══════════════════════════════════════════════════════════════════

1. ⚠️ ALWAYS change default password (admin123) immediately!

2. 🔐 Use Solution 2 with strong custom credentials for production

3. 🚫 Don't share admin credentials publicly

4. 🔄 Rotate admin password regularly

5. 🛡️ After setup, disable setup page:
   Render Environment → Add: DISABLE_SETUP=True

6. 📧 Use a real email for admin (for password reset)

7. 🔑 Use strong password:
   - At least 12 characters
   - Mix of letters, numbers, symbols
   - Not a dictionary word
